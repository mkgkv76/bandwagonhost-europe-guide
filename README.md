## 三个欧洲机房，差在哪里

搬瓦工的欧洲机房名字有点绕，但其实对应逻辑很简单：

**EUNL_1（荷兰 DC1）——三网全顶配，买的时候要注意**

这是目前搬瓦工欧洲机房里线路最好的一个。电信走 CN2 GIA，联通走 AS9929，移动走 CMIN2，三网全都是各自最高端的回程路由。硬件也换成了 AMD EPYC Genoa（Zen4 架构）加 NVMe SSD，带宽直接给到 10Gbps。

实测延迟方面，移动和联通大概在 130ms 上下，电信稍微高一些在 180-210ms 左右，但丢包率极低，全天 0% 丢包的表现在欧洲机房里算是罕见。

有一个坑要提前说：**EUNL_1 机房只能在购买 CN2 GIA-E 套餐的时候选择，买完之后如果迁移到其他机房，就再也迁不回来了。** 不是临时换换机房那种操作——迁出去就是永久失去 EUNL_1。所以如果真想用这个机房，下单的时候就要想清楚，直接选好，后面不要乱动。

**EUNL_9（荷兰 DC9）——联通用户的首选欧洲机房**

EUNL_9 的定位是联通高端线路机房。回程接入联通 AS9929 和 AS10099，对联通用户来说三网延迟都能控制在 200ms 以内，算是欧洲机房里对国内用户最友好的一档。

电信和移动用 EUNL_9 也能跑，因为三网回程都经过联通高端线路，不会走普通路由。实测下载速度大概在 400Mbps 上下，丢包率总体偏低。不过和 EUNL_1 相比，硬件还是上一代，单核性能有差距。

目前购买 CN2 GIA-E 套餐之后，可以在 KiwiVM 后台自由迁移到 EUNL_9，不像 EUNL_1 那样有限制。这也是大多数人选欧洲机房的主要路径。

**EUNL_2（荷兰 DC2）——入门机，普通线路**

EUNL_2 是最普通的那个。线路没有额外优化，用的是普通 BGP 接入，对国内用户来说延迟会更高，也没有 CN2 或联通高端的加持。

KVM 套餐和 CN2 GIA-E 套餐都可以用 EUNL_2，所以主要是面向欧洲本地访客、不需要优化国内回程的场景，比如纯面向欧洲用户的外贸独立站。

说实话，搬瓦工欧洲机房面向的需求挺具体的，不是所有人都需要。

对于纯粹的国内访问需求（比如国内用户访问的中文网站），洛杉矶 DC6 或者日本软银机房还是更合适，物理距离更近，延迟更低。
